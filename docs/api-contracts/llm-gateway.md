# LlmGateway / Adapter Internal Contract

Owner: BE-A

## Internal Interface

```text
LlmGatewayResponse review(LlmGatewayRequest request)
```

Request fields:

```text
providerId required
modelName optional, overrides provider.defaultModel
messages required, OpenAI-compatible role/content messages
```

Response fields:

```text
status
rawResponse
contentText
structuredJson
latencyMs
errorCode
errorMessage
```

## Status Values

```text
SUCCESS
PROVIDER_UNAVAILABLE
PROVIDER_ERROR
TIMEOUT
INVALID_JSON
```

Rules:

```text
Only enabled providers can be called.
Provider API key is decrypted only for the runtime call and is never returned.
Adapter sends POST {baseUrl}/chat/completions.
Provider custom headers are injected before Authorization is set.
Authorization is always normalized to Bearer <apiKey> when an API key exists.
Raw provider response is retained in rawResponse for troubleshooting and later Agent Run persistence.
```

## Structured JSON Extraction

```text
Gateway reads choices[0].message.content from the OpenAI-compatible response.
Content can be pure JSON or Markdown fenced JSON.
If content is not valid JSON, status=INVALID_JSON and rawResponse/contentText are preserved.
```

## Error Mapping

```text
Missing or disabled provider -> PROVIDER_UNAVAILABLE
Provider HTTP non-2xx or I/O failure -> PROVIDER_ERROR
Adapter timeout -> TIMEOUT
OpenAI-compatible body or model content cannot produce structured JSON -> INVALID_JSON
```

## Configuration

```text
labelhub.llm.gateway.timeout-ms
Environment override: LABELHUB_LLM_GATEWAY_TIMEOUT_MS
Default: 30000
```

Security:

```text
API Key must not appear in errorMessage, audit logs, application logs, rawResponse generated by backend, or public API responses.
```
