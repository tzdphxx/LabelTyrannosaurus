# FE 前端开发任务书

> 面向 AI 开发工具：本文件只描述前端负责范围。开发时优先阅读本文件，再按需查阅 `04\-对接契约与数据结构协作手册\.md`。不要在前端任务中擅自改后端契约。

## 0\. 前端目标

前端负责把 LabelHub 做成完整可演示的多角色 Web 工作台：

- Admin 能管理用户、角色、模型厂商。

- Owner 能建任务、导入数据、批量编辑、搭模板、配置 AI、发布、导出。

- Labeler 能领取任务、作答、保存草稿、提交、查看打回和贡献。

- Reviewer 能审核、批量操作、处理冲突、选择金标。

核心要求：

- 页面不要只做壳子，必须跑通真实用户路径。

- 所有表格、表单、详情页必须有 loading、empty、error。

- 所有长流程页面必须能从错误中恢复。

- 断网或接口超时不能导致用户输入丢失。

## 1\. 前端基础工程

### 1\.1 大功能：应用骨架

交付效果：

- 用户打开应用后，根据登录态和角色进入正确工作台。

- 没有权限的入口不可见。

- 接口 401 时自动跳登录。

小功能：

1. 路由结构

    - `/login`

    - `/register`

    - `/admin`

    - `/owner`

    - `/labeler`

    - `/reviewer`

2. Layout

    - 顶部用户信息。

    - 左侧角色菜单。

    - 内容区。

3. 登录态

    - `authStore` 保存 token、用户、角色。

    - 刷新页面后恢复登录态。

4. API Client

    - 自动带 token。

    - 统一处理 code/message/traceId。

    - 401 清理登录态。

验收：

```Plaintext
未登录访问业务页跳转 /login。
Labeler 看不到 Owner / Reviewer / Admin 菜单。
Admin 能进入管理页。
接口错误能显示 traceId。
```

### 1\.2 大功能：通用组件

交付效果：

- 后续页面使用统一组件，减少重复。

小功能：

1. `PageHeader`

    - 标题。

    - 描述。

    - 操作按钮区。

2. `DataTable`

    - loading。

    - empty。

    - pagination。

    - row selection。

3. `ResultPanel`

    - 批量操作成功数。

    - 失败数。

    - 失败原因列表。

4. `StatusTag`

    - 任务状态。

    - 提交状态。

    - AI 审核状态。

5. `JsonPreview`

    - 格式化 JSON。

    - 支持折叠。

6. `AuditTimeline`

    - 展示审计事件。

验收：

```Plaintext
所有列表页使用统一 DataTable。
所有状态用 StatusTag。
批量操作统一用 ResultPanel 展示结果。
```

    - baseUrl。

    - apiKey。
