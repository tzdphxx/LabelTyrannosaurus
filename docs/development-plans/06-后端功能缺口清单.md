## 三、端到端流程阻塞分析

### Owner 创建任务流程

```
1. ✅ 创建任务 → POST /api/v1/tasks
2. ❌ 导入数据集 → POST /api/v1/tasks/{taskId}/dataset/import [缺失]
3. ❌ 创建模板 → POST /api/v1/tasks/{taskId}/templates [缺失]
4. ❌ 配置奖励 → POST /api/v1/tasks/{taskId}/reward-rule [缺失]
5. ✅ 配置 AI 审核 → POST /api/v1/tasks/{taskId}/ai-review-configs
6. ✅ 发布任务 → POST /api/v1/tasks/{taskId}/publish
```

**结论：** 步骤 2/3/4 缺失，Owner 无法完成完整的任务创建流程。

### Labeler 标注流程

```
1. ✅ 浏览任务广场 → GET /api/v1/market/tasks
2. ✅ 领取任务 → POST /api/v1/tasks/{taskId}/assignments/claim
3. ✅ 保存草稿 → PUT /api/v1/assignments/{assignmentId}/draft
4. ✅ 提交答案 → POST /api/v1/assignments/{assignmentId}/submit
5. ❌ 查看提交历史 → GET /api/v1/labeler/submissions [缺失]
6. ❌ 查看贡献统计 → GET /api/v1/labeler/contribution/overview [缺失]
```

**结论：** 核心标注流程可用，但 Labeler 无法查看自己的提交历史和贡献。

### Reviewer 审核流程

```
1. ✅ 待审列表 → GET /api/v1/reviewer/submissions
2. ❌ 提交详情 → GET /api/v1/reviewer/submissions/{submissionId} [缺失]
3. ✅ 通过/打回 → POST /api/v1/reviewer/submissions/{submissionId}/approve|reject
4. ✅ 批量操作 → POST /api/v1/reviewer/submissions/batch/*
5. ✅ 冲突仲裁 → POST /api/v1/reviewer/conflict-groups/{groupId}/resolve
6. ❌ 导出数据 → POST /api/v1/tasks/{taskId}/exports [缺失]
```

**结论：** 审核流程基本可用，但 Reviewer 无法查看单条提交详情（含 AI 结果）。

## 四、实现优先级建议

### P0（阻塞前端联调，必须优先实现）

1. **数据集导入** — 无数据集则无法发布任务
2. **模板创建** — 无模板则无法发布任务
3. **奖励规则配置** — 无奖励规则则无法发布任务
4. **导出任务管理** — Owner 需要导出标注结果
5. **Assignment 详情查询** — Labeler 工作台需要
6. **Submission 详情查询** — Reviewer 审核需要
7. **Labeler 提交历史** — Labeler 需要查看自己的提交

### P1（功能完整性，可延后）

- 数据集批量追加/更新/删除
- Schema 校验端点
- 贡献统计（overview/trend/tasks）
- 奖励流水
- AI 审核结果查询

### P2（增强功能，最后实现）

- 数据集覆盖导入
- 模板复制
- 手动触发 AI 重试
- Agent Run 详情查询
